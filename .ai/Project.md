# GymTracker Admin

## Objetivo

Interface administrativa para manutenção do catálogo.

Todos os dados são persistidos através da GymTracker API.

---

# Stack

Framework

- Next.js 15

Linguagem

- TypeScript

UI

- TailwindCSS
- Shadcn UI

Formulários

- React Hook Form
- Zod

Requisições

- Axios

Cache

- TanStack Query

---

# Regras

Toda comunicação deve ocorrer via API.

Não acessar banco diretamente.

---

# Arquitetura

Feature First

Cada módulo deve conter:

- Components
- Hooks
- Services
- Schemas
- Types
